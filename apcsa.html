Primitive Types & Objects

Data types, variables, expressions, operators, string manipulation, Math class, and input/output fundamentals.

~10–15% of AP Exam

Primitive Data Types

  • int — whole numbers (−2³¹ to 2³¹−1)
  • double — floating-point numbers
  • boolean — true or false
  • char — single character (Unicode)
  • Stored directly in memory (stack)
  • Default values: 0, 0.0, false, '\u0000'

Reference Types & Objects

  • String, Scanner, Integer, Double
  • Store a reference (address) to heap memory
  • Default value is null
  • Created with new keyword (usually)
  • Accessed through methods
  • String literals are special — interned

Operators & Expressions

  • Arithmetic: + − * / % (integer division truncates)
  • Compound assignment: += −= *= /= %=
  • Increment/decrement: ++ and −−
  • Operator precedence: * / % before + −
  • Integer division: 7/2 = 3 (not 3.5)
  • Casting: (double) 7 / 2 = 3.5

String Methods (AP Subset)

  • length() — number of characters
  • substring(a, b) — chars from index a to b−1
  • indexOf(s) — first occurrence of s (−1 if not found)
  • equals(s) / equalsIgnoreCase(s)
  • compareTo(s) — lexicographic comparison
  • Strings are immutable in Java

1.1 Variables, Types & Declaration

In Java, every variable has a declared type. Primitive types store values directly; reference types store addresses. Variables must be declared before use and have a defined scope.

Sample Questions MCQFRQ

MCQ What is the output of: System.out.println("val: " + 3 + 4); ▼

Consider the following statement:

System.out.println("val: " + 3 + 4);
What is printed?

  • A val: 7
  • B val: 34
  • C val: 3 + 4
  • D Compile error
✓ CORRECT ANSWER: B

The + operator is evaluated left to right. First, "val: " + 3 produces the String "val: 3" (String concatenation). Then "val: 3" + 4 produces "val: 34". There is no addition of integers here because once a String is encountered, subsequent + operations become concatenation. To get 7, you would need: "val: " + (3 + 4).

MCQ Integer division and modulo result ▼

What are the values of x and y after the following executes?

int x = 17 / 5;
int y = 17 % 5;

  • A x = 3.4, y = 2
  • B x = 3, y = 2
  • C x = 3, y = 17
  • D x = 4, y = 2
✓ CORRECT ANSWER: B

Integer division in Java truncates (not rounds) toward zero: 17 ÷ 5 = 3 remainder 2, so 17 / 5 = 3 (decimal part discarded). The modulo operator % gives the remainder: 17 = 5×3 + 2, so 17 % 5 = 2. A is wrong because int cannot hold 3.4. C is wrong (17 would mean % wasn't computed). D is wrong (5 × 4 = 20 > 17).

MCQ String substring indexing ▼

What is the result of the following?

String s = "Programmers";
System.out.println(s.substring(3, 7));

  • A "gram"
  • B "ramm"
  • C "gram"
  • D "rogr"
✓ CORRECT ANSWER: A/C — "gram"

Java's substring(start, end) returns characters from index start up to (but not including) index end. "Programmers" has indices: P=0, r=1, o=2, g=3, r=4, a=5, m=6, m=7, e=8, r=9, s=10. So substring(3, 7) = characters at 3,4,5,6 = "gram". The key rule: end index is exclusive.

FRQ FRQ: String Manipulation — encodeMessage ▼

Write a static method encodeMessage that takes a String and returns a new String where every word is reversed and all vowels (a, e, i, o, u — case-insensitive) are replaced with an asterisk *.

Part A: Write a helper method replaceVowels(String word) that returns the word with all vowels replaced by *.
Part B: Write reverseWord(String word) that returns the word reversed.
Part C: Write encodeMessage(String message) that splits on spaces, reverses each word, replaces vowels, and joins with spaces.
◆ SOLUTION
Solution.java 
// Part A: replace vowels
public static String replaceVowels(String word) {
    String result = "";
    String vowels = "aeiouAEIOU";
    for (int i = 0; i < word.length(); i++) {
        char c = word.charAt(i);
        if (vowels.indexOf(c) != -1)
            result += "*";
        else
            result += c;
    }
    return result;
}

// Part B: reverse a word
public static String reverseWord(String word) {
    String rev = "";
    for (int i = word.length() - 1; i >= 0; i--)
        rev += word.charAt(i);
    return rev;
}

// Part C: encode full message
public static String encodeMessage(String message) {
    String[] words = message.split(" ");
    String encoded = "";
    for (int i = 0; i < words.length; i++) {
        if (i > 0) encoded += " ";
        encoded += replaceVowels(reverseWord(words[i]));
    }
    return encoded;
}
// encodeMessage("Hello World") → "*ll*H dl*W"
✓ EXPLANATION

Part A: Iterate through each character; use indexOf on a vowel-string to check membership in O(1) logic. Build result with concatenation.

Part B: Loop from the last index (length()-1) down to 0, appending each char.

Part C: Split on spaces, then apply both helpers to each token. Note the order: reverse first, then replace vowels (or vice versa — both are independent operations). Handle spacing by checking if i > 0 before adding a space.

Writing Classes

Object-oriented design: constructors, instance variables, methods, encapsulation, static vs. instance, boolean logic, conditionals, and iteration.

~25–30% of AP Exam

Class Anatomy

  • Instance variables (fields) — describe state
  • Constructors — initialize state (no return type)
  • Accessor (getter) methods — return state
  • Mutator (setter) methods — modify state
  • Static fields/methods — belong to the class
  • this keyword — refers to current object

Encapsulation & Access

  • private fields — only accessible within the class
  • public methods — accessible everywhere
  • Encapsulation hides implementation details
  • Prevents invalid state through validation
  • toString() — human-readable representation
  • equals() — value equality comparison

Boolean & Conditionals

  • Relational: == != < > <= >=
  • Logical: && (and), || (or), ! (not)
  • Short-circuit evaluation
  • if / else if / else chains
  • Nested conditionals
  • De Morgan's Laws: !(A && B) = !A || !B

Iteration

  • while loop — condition checked before each iteration
  • for loop — classic counter-based iteration
  • do-while — executes body at least once
  • Nested loops — O(n²) or O(n·m) complexity
  • break / continue — loop control
  • Common patterns: accumulator, counter, search

2.1 Designing a Class

BankAccount.java 
public class BankAccount {
    // Instance variables — private for encapsulation
    private String owner;
    private double balance;
    private static int totalAccounts = 0; // class variable

    // Constructor
    public BankAccount(String owner, double initialBalance) {
        this.owner = owner;
        this.balance = (initialBalance > 0) ? initialBalance : 0;
        totalAccounts++;
    }

    // Accessor (getter)
    public double getBalance() { return balance; }
    public String getOwner()  { return owner;   }

    // Mutator with validation
    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public boolean withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false; // insufficient funds
    }

    // Static method
    public static int getTotalAccounts() { return totalAccounts; }

    @Override
    public String toString() {
        return owner + ": $" + balance;
    }
}

2.2 Boolean Logic & De Morgan's Laws

De Morgan's Laws are essential for simplifying boolean expressions on the AP exam:

OriginalDe Morgan's EquivalentPlain English
!(A && B)!A || !BNOT (both) = (not A) OR (not B)
!(A || B)!A && !BNOT (either) = (not A) AND (not B)
✅

Short-circuit evaluation: In A && B, if A is false, B is never evaluated. In A || B, if A is true, B is never evaluated. This matters for avoiding NullPointerExceptions: obj != null && obj.method().

2.3 Loop Patterns

Loops.java 
// Pattern 1: Accumulator
int sum = 0;
for (int i = 1; i <= 10; i++) sum += i; // sum = 55

// Pattern 2: Counting matches
int count = 0;
for (int i = 1; i <= 20; i++)
    if (i % 3 == 0) count++;

// Pattern 3: while with unknown iterations
int n = 100;
while (n > 1) {
    if (n % 2 == 0) n /= 2;
    else            n = 3 * n + 1; // Collatz conjecture
}

// Pattern 4: Nested loops — multiplication table
for (int r = 1; r <= 3; r++) {
    for (int c = 1; c <= 3; c++)
        System.out.print(r * c + "\t");
    System.out.println();
}

Sample Questions MCQFRQ

MCQ What does a constructor return? ▼

Which of the following is true about constructors in Java?

  • A A constructor must return void
  • B A constructor can return any type
  • C A constructor has no return type (not even void)
  • D A constructor always returns a reference to a new object
✓ CORRECT ANSWER: C

Constructors have no return type declaration at all — not even void. They are named exactly like the class. Writing a return type would make it a regular method with the same name as the class, not a constructor. The new keyword is what actually causes the object to be created and returned — the constructor simply initializes it. D describes what new does, not the constructor's declared return type.

MCQ Tracing a nested loop ▼

How many times does System.out.print("*") execute?

for (int i = 1; i <= 4; i++)
for (int j = 1; j <= i; j++)
System.out.print("*");

  • A 8
  • B 12
  • C 10
  • D 16
✓ CORRECT ANSWER: C

Trace the inner loop for each value of i: when i=1, j goes 1..1 → 1 print; i=2, j goes 1..2 → 2 prints; i=3, j goes 1..3 → 3 prints; i=4, j goes 1..4 → 4 prints. Total: 1+2+3+4 = 10. This is a triangle pattern. The outer loop runs 4 times, but the inner loop runs i times each iteration, not a fixed number.

FRQ FRQ: Writing a Class — Student class design ▼

Write a complete Java class Student that models a student with a name, a GPA (0.0 to 4.0), and a grade level (9–12). Include appropriate constructors, accessors, and mutators with validation. Include a method isHonors() that returns true if GPA ≥ 3.5 and a toString() method.

◆ SOLUTION
Student.java 
public class Student {
    private String name;
    private double gpa;
    private int gradeLevel;

    // Constructor with validation
    public Student(String name, double gpa, int gradeLevel) {
        this.name = name;
        this.gpa = (gpa >= 0.0 && gpa <= 4.0) ? gpa : 0.0;
        this.gradeLevel = (gradeLevel >= 9 && gradeLevel <= 12)
                           ? gradeLevel : 9;
    }

    // Accessors
    public String getName()     { return name; }
    public double getGPA()      { return gpa; }
    public int    getGradeLevel() { return gradeLevel; }

    // Mutators with validation
    public void setGPA(double gpa) {
        if (gpa >= 0.0 && gpa <= 4.0) this.gpa = gpa;
    }

    public void setGradeLevel(int level) {
        if (level >= 9 && level <= 12) this.gradeLevel = level;
    }

    // Business logic
    public boolean isHonors() { return gpa >= 3.5; }

    @Override
    public String toString() {
        return name + " (Grade " + gradeLevel + ", GPA: "
               + gpa + ")"
               + (isHonors() ? " [Honors]" : "");
    }
}
✓ EXPLANATION — Key AP Scoring Points

• Private fields — encapsulation requires private instance variables.
• Validation in constructor and setter — use a ternary or if-statement; invalid values should revert to a safe default (not crash).
• this keyword — required when parameter name matches field name to disambiguate.
• isHonors() — a simple boolean method returning a comparison expression. No need for if/else return true/false.
• toString() — must have exactly this signature (public String toString()) to override Object's toString properly.

Arrays & ArrayLists

1D and 2D arrays, ArrayList class, iteration patterns, searching, sorting fundamentals, and algorithm analysis.

~25–30% of AP Exam

Arrays

  • Fixed size — set at creation, can't change
  • Zero-indexed: valid range 0 to length−1
  • Default values: 0, 0.0, false, null
  • arr.length — NOT a method call (no ())
  • ArrayIndexOutOfBoundsException if out of range
  • for-each loop: for (Type x : arr)

ArrayList

  • Dynamic size — grows and shrinks automatically
  • import java.util.ArrayList;
  • Only holds objects (use Integer, Double, etc.)
  • add(e), add(i,e), remove(i), get(i), set(i,e)
  • size() — method (with ())
  • Auto-boxing: int ↔ Integer automatically

2D Arrays

  • int[][] grid = new int[rows][cols]
  • grid.length → number of rows
  • grid[0].length → number of columns
  • Row-major traversal: outer=rows, inner=cols
  • Column-major: outer=cols, inner=rows
  • Can be irregular (jagged) arrays

Searching & Sorting

  • Sequential search — O(n), works on any list
  • Binary search — O(log n), needs sorted array
  • Selection sort — find min, swap to front, O(n²)
  • Insertion sort — build sorted portion, O(n²)
  • Merge sort / quicksort — O(n log n)
  • Arrays.sort() and Collections.sort() built-in

3.1 Array Fundamentals

ArrayDemo.java 
// Declaration and initialization
int[] scores = {85, 92, 78, 96, 88};
int[] empty  = new int[10];   // all zeros

// Traversal patterns
for (int i = 0; i < scores.length; i++)
    System.out.print(scores[i] + " ");

// Enhanced for loop (read-only)
for (int s : scores)
    System.out.print(s + " ");

// Common algorithm: find maximum
int max = scores[0];  // start with first, not 0!
for (int i = 1; i < scores.length; i++)
    if (scores[i] > max) max = scores[i];
// max = 96

// Reverse traversal
for (int i = scores.length - 1; i >= 0; i--)
    System.out.print(scores[i] + " ");

3.2 ArrayList Operations

ArrayListDemo.java 
import java.util.ArrayList;

ArrayList<String> names = new ArrayList<>();
names.add("Alice");      // ["Alice"]
names.add("Bob");        // ["Alice","Bob"]
names.add(0, "Zara");    // ["Zara","Alice","Bob"]
names.remove(1);         // ["Zara","Bob"]  (removes "Alice")
names.set(0, "Xena");    // ["Xena","Bob"]
names.get(1);            // "Bob"
names.size();             // 2

// DANGER: removing while iterating forward — indices shift!
// CORRECT: traverse backwards when removing
ArrayList<Integer> nums = new ArrayList<>();
for (int i = 1; i <= 5; i++) nums.add(i);

for (int i = nums.size() - 1; i >= 0; i--)
    if (nums.get(i) % 2 == 0)
        nums.remove(i); // removes evens: [1,3,5]
⚠️

Index shifting trap: When you remove element at index i while looping forward, the next element shifts to index i and gets skipped. Always loop backwards when removing from an ArrayList during iteration.

Sample Questions MCQFRQ

MCQ ArrayList removal during forward iteration ▼

An ArrayList contains [2, 4, 4, 6, 8]. The following code is intended to remove all 4s:

for (int i = 0; i < list.size(); i++)
if (list.get(i) == 4)
list.remove(i);
What is in the ArrayList after this code runs?

  • A [2, 6, 8]
  • B [2, 4, 6, 8]
  • C [4, 6, 8]
  • D [2, 6, 4, 8]
✓ CORRECT ANSWER: B

When i=1, list.get(1)=4, so it's removed. The list becomes [2,4,6,8] and i becomes 2. Now list.get(2)=6 (the second 4 shifted to index 1 but i skipped it!). The loop continues and never revisits index 1. Result: one 4 remains at index 1. This is the classic "index shift" bug. The fix is to loop backwards or decrement i after removal.

MCQ 2D array traversal — column sum ▼

Given: int[][] m = {{1,2,3},{4,5,6},{7,8,9}};
What does this code print?

int s = 0;
for (int r = 0; r < m.length; r++)
s += m[r][1];
System.out.println(s);

  • A 6
  • B 15
  • C 12
  • D 45
✓ CORRECT ANSWER: B

The code sums column 1 (the middle column, index 1) across all rows. m[0][1]=2, m[1][1]=5, m[2][1]=8. Sum = 2+5+8 = 15. Note: m[r][1] fixes the column at 1 and varies the row — this is a column traversal pattern. Answer A (6) would be the sum of row 0. Answer D (45) would be all elements. Answer C (12) would be column 2 (third column): 3+6+3... wait, that's wrong too — C is a distractor.

FRQ FRQ: Array Processing — Statistics methods ▼

Write methods for a statistics utility class that processes an array of doubles. Assume the array has at least one element.

Part A: Write mean(double[] data) — returns the arithmetic mean.
Part B: Write countAbove(double[] data, double threshold) — returns the count of elements strictly greater than threshold.
Part C: Write normalize(double[] data) — returns a new array where each element is (element − mean) / max(1, standardRange). StandardRange = max − min of data. If max equals min, return all zeros.
◆ SOLUTION
Statistics.java 
// Part A: mean
public static double mean(double[] data) {
    double sum = 0;
    for (double v : data) sum += v;
    return sum / data.length;
}

// Part B: count above threshold
public static int countAbove(double[] data, double threshold) {
    int count = 0;
    for (double v : data)
        if (v > threshold) count++;
    return count;
}

// Part C: normalize
public static double[] normalize(double[] data) {
    double min = data[0], max = data[0];
    for (double v : data) {
        if (v < min) min = v;
        if (v > max) max = v;
    }
    double[] result = new double[data.length];
    if (max == min) return result; // all zeros
    double avg = mean(data);
    double range = max - min;
    for (int i = 0; i < data.length; i++)
        result[i] = (data[i] - avg) / range;
    return result;
}
✓ EXPLANATION

• Part A: Accumulator pattern with enhanced for-loop. Divide by length (cast is not needed since sum is already double).

• Part B: Counter pattern. Use strict greater-than as specified ("strictly greater than").

• Part C: Initialize min and max from index 0, not Integer.MIN/MAX (good practice for doubles). Check max==min before dividing to avoid division by zero. Reuse the mean() method from Part A — the AP exam rewards method reuse. Return a new array to avoid mutating the input.

Inheritance & Recursion

Superclasses and subclasses, polymorphism, abstract classes, interfaces, and recursive algorithms.

~20–25% of AP Exam

Inheritance Basics

  • extends keyword creates a subclass (IS-A)
  • Subclass inherits public/protected members
  • super() — call superclass constructor (first line)
  • super.method() — call superclass method
  • Cannot inherit private fields directly
  • Every class implicitly extends Object

Polymorphism & Overriding

  • Override: same signature, different behavior
  • @Override annotation — catches mistakes
  • Dynamic dispatch — runtime type determines method
  • Overload: same name, different parameters
  • instanceof — check runtime type
  • Object type can hold subclass reference

Abstract Classes & Interfaces

  • abstract class — cannot be instantiated
  • abstract method — subclass must implement
  • Interface — contract, all methods public abstract
  • implements keyword for interfaces
  • A class can implement multiple interfaces
  • Comparable interface — compareTo() method

Recursion

  • Base case — stops recursion (required!)
  • Recursive case — calls itself with simpler input
  • Call stack — each call gets its own frame
  • StackOverflowError if no base case
  • Binary search is recursive naturally
  • Merge sort uses divide-and-conquer recursion

4.1 Inheritance Hierarchy

Shape.java → Circle.java 
// Superclass
public abstract class Shape {
    private String color;

    public Shape(String color) { this.color = color; }

    public String getColor() { return color; }

    public abstract double area();   // subclass must implement

    @Override
    public String toString() {
        return color + " shape, area=" + area();
    }
}

// Subclass
public class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);  // MUST be first statement!
        this.radius = radius;
    }

    @Override
    public double area() { return Math.PI * radius * radius; }

    @Override
    public String toString() {
        return super.toString() + ", r=" + radius;
    }
}

// Polymorphism in action
Shape s = new Circle("red", 5); // Shape ref, Circle obj
s.area();    // calls Circle.area() — dynamic dispatch!
// s.radius → COMPILE ERROR (Shape doesn't have radius)

4.2 Recursion — Thinking Recursively

Every recursive method needs: (1) a base case that returns without a recursive call, and (2) a recursive case that makes the problem smaller. Trust that the recursive call works — just ensure the base case and the reduction are correct.

Recursion.java 
// Classic: factorial
public static int factorial(int n) {
    if (n == 0 || n == 1) return 1;  // base case
    return n * factorial(n - 1);       // recursive case
}
// factorial(4) → 4 * factorial(3) → 4*3*2*1 = 24

// Fibonacci (two recursive calls)
public static int fib(int n) {
    if (n <= 1) return n;              // base cases: fib(0)=0, fib(1)=1
    return fib(n-1) + fib(n-2);       // recursive case
}

// Recursive binary search
public static int binarySearch(int[] arr, int target,
                                 int lo, int hi) {
    if (lo > hi) return -1;                  // base: not found
    int mid = (lo + hi) / 2;
    if (arr[mid] == target) return mid;  // base: found
    if (arr[mid] < target)                 // search right
        return binarySearch(arr, target, mid+1, hi);
    else                                     // search left
        return binarySearch(arr, target, lo, mid-1);
}
💡

Tracing recursion on the AP exam: Draw a call tree. Start from the top call, expand children until you hit base cases, then bubble results back up. For factorial(4): 4→3→2→1(return 1)→2(return 2)→3(return 6)→4(return 24).

Sample Questions MCQFRQ

MCQ Polymorphism — which method is called? ▼

Given the following class hierarchy where Animal has a sound() method returning "...", and Dog extends Animal and overrides sound() to return "Woof":

Animal a = new Dog();
System.out.println(a.sound());
What is printed?

  • A "..."
  • B "Woof"
  • C Compile error
  • D Runtime error
✓ CORRECT ANSWER: B

Dynamic dispatch (runtime polymorphism) means Java uses the actual runtime type of the object to determine which method to call — not the declared reference type. The variable a is declared as Animal but holds a Dog object. Since Dog overrides sound(), Dog's version is called. This only works for overridden instance methods, not for fields or static methods (those use the declared type).

MCQ Tracing a recursive method ▼

What does mystery(5) return?

public static int mystery(int n) {
if (n <= 0) return 0;
return n + mystery(n - 2);
}

  • A 15
  • B 9
  • C 6
  • D Infinite recursion
✓ CORRECT ANSWER: B

Trace the calls:
mystery(5) = 5 + mystery(3)
mystery(3) = 3 + mystery(1)
mystery(1) = 1 + mystery(-1)
mystery(-1) = 0   ← base case (n ≤ 0)

Unwind: mystery(-1)=0, mystery(1)=1+0=1, mystery(3)=3+1=4, mystery(5)=5+4=9.
This sums odd numbers ≤ n. D is wrong because the base case n≤0 is always eventually reached (n decreases by 2 each call, eventually crossing 0).

FRQ FRQ: Inheritance — Vehicle class hierarchy ▼

Design an inheritance hierarchy for vehicles. A Vehicle has a make, model, and year. An ElectricVehicle extends Vehicle and adds a battery capacity (kWh) and range (miles). Write both classes with appropriate constructors and methods.

Part A: Write the Vehicle superclass with getters and a toString().
Part B: Write ElectricVehicle with a constructor that calls super(), a method efficiency() returning miles-per-kWh, and override toString() to include battery info.
◆ SOLUTION
Vehicle.java + ElectricVehicle.java 
// Part A: Superclass
public class Vehicle {
    private String make;
    private String model;
    private int    year;

    public Vehicle(String make, String model, int year) {
        this.make  = make;
        this.model = model;
        this.year  = year;
    }

    public String getMake()  { return make;  }
    public String getModel() { return model; }
    public int    getYear()  { return year;  }

    @Override
    public String toString() {
        return year + " " + make + " " + model;
    }
}

// Part B: Subclass
public class ElectricVehicle extends Vehicle {
    private double batteryCapacity; // kWh
    private double range;           // miles

    public ElectricVehicle(String make, String model, int year,
                            double battery, double range) {
        super(make, model, year); // call superclass constructor
        this.batteryCapacity = battery;
        this.range = range;
    }

    public double getBattery() { return batteryCapacity; }
    public double getRange()   { return range; }

    public double efficiency() {
        return range / batteryCapacity; // miles per kWh
    }

    @Override
    public String toString() {
        return super.toString()           // reuse parent
               + " [EV, " + batteryCapacity + "kWh, "
               + range + "mi range]";
    }
}
// new ElectricVehicle("Tesla","Model3",2024,82,350).toString()
// → "2024 Tesla Model3 [EV, 82.0kWh, 350.0mi range]"
✓ EXPLANATION — AP Scoring Priorities

• super() first: The superclass constructor call must be the very first line of the subclass constructor. Omitting it or placing it later is a compile error.

• Private inheritance: Because Vehicle's fields are private, ElectricVehicle cannot access them directly — it must use the getters (getMake(), getModel(), etc.). This is correct encapsulation.

• super.toString(): Reusing the parent's toString() in the override avoids duplicating logic and is explicitly rewarded by AP graders.

• @Override: Always include this annotation — it catches signature mistakes at compile time and signals intent to graders.

• efficiency(): A subclass-specific method unavailable through a Vehicle reference — only usable when the reference type is ElectricVehicle.

FRQ FRQ: Recursion — palindrome checker ▼

Write a recursive method isPalindrome(String s) that returns true if the string is a palindrome (reads the same forwards and backwards). Do not use loops. Ignore case.

Base cases: A string of length 0 or 1 is always a palindrome.
Recursive case: Check if the first and last characters match (ignoring case), and if so, recurse on the middle substring.
◆ SOLUTION
Palindrome.java 
public static boolean isPalindrome(String s) {
    // Base cases: empty or single character
    if (s.length() <= 1) return true;

    // Get first and last characters (ignore case)
    char first = Character.toLowerCase(s.charAt(0));
    char last  = Character.toLowerCase(s.charAt(s.length() - 1));

    // If outer chars don't match, not a palindrome
    if (first != last) return false;

    // Recurse on the middle substring
    return isPalindrome(s.substring(1, s.length() - 1));
}

// isPalindrome("Racecar") → true
// isPalindrome("hello")   → false
// isPalindrome("A")       → true
// isPalindrome("")        → true
✓ EXPLANATION

• Two base cases: Empty string and single character are both palindromes trivially. Combining with <= 1 handles both in one check.

• Recursive reduction: substring(1, length-1) strips one character from each end, making the problem strictly smaller each call. This guarantees we reach the base case.

• Early return false: If the outer characters don't match, we immediately return false without making an unnecessary recursive call — this is an optimization the AP exam rewards.

• Case handling: Converting to lowercase before comparing handles the ignore-case requirement cleanly.

• No loops: This solution is purely recursive — each call makes exactly one recursive call or returns a base case value.
